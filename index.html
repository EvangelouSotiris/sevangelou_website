Asynxy: Asynchronous Proxy RCE

Asynxy is a "middleware" web application that performs remote code execution for a "master" node to a set of slaves. This is achieved through a periodic request for new commands from the slaves, a local execution using Python and lastly a post of the command's results back to the webapp for the master to see. The benefits that the webapp introduces is control over a large set of slaves either individually or as a group (botnet-like) and a connection mostly unbothered by firewalls or NAT since it is based on HTTP and is always initiated from the slaves.

Real time Network IDS with Machine Learning

This project was the final project for two courses: ECE417 and ECE514. For the ECE417 we created a Network Intrusion Detection System that classifies on the normality of the traffic, detecting possible cyber attacks happening. We experimented on Random Forest, Support Vector Machine and Multilayer Perceptron with the latter being the best-performing ML model on data taken from the Australian Center for Cyber Security (UNSW). For ECE514 we focused on creating a "pipeline" that collects data in real time, extracts features to create datasets and passes them to our MLP model for live predictions. The repository contains - apart from the code - the paper reports and presentations of the project.

tASE: Automated Security Assessments

tASE is a collaborative project with GRNET CERT. It is a framework that automates platform or application security testing through the use of various security tools using user defined scenarios written in Gherkin format. It can do both static and dynamic analysis using tools such as OWASP ZAP, Nmap, Bandit. The project has also well-defined ways to add new tools and write more complex scenarios, thus leveraging End User Development. The project is not Open Source.


Network Protocol Attacks Study

Project done for the course Internet Protocol Design. The project delves into popular Network attacks on network protocols, specifically ARP and DNS Spoofing, MAC Table Overflow, TCP SYN Flooding, TCP RST attack, TCP Session Highjacking and Heartbleed. We set up test environments where we used tools and scripts to perform these attacks on test victims, and we created a report containing the theory behind the attacks, and the steps we followed to perform them.

Common Vulnerabilities of Ethereum Smart Contracts

For the Blockchain course, we studied Ethereum smart contracts written in Solidity, we tried to identify common Security Vulnerabilities that led to serious exploits (e.g. DAO Hack) and bad coding practices, and we presented a proof of concept live demo on some of them after first testing vulnerable code with Oyente (smart contract static analysis tool), then compiling it deploying it into the Ropsten test network and exploiting them within the online IDE Remix.
